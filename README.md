# QDHub

一个基于 Go 开发的量化数据管理系统，用于管理数据源的元数据，实现自动建表和量化数据的自动同步、管理、分析和可视化。

## 项目简介

QDHub 是一个面向量化交易的数据管理平台，提供了从数据源元数据管理到数据同步、存储、分析和可视化的完整解决方案。

## 主要功能

- **元数据管理**：管理各类数据源的元数据信息
- **自动建表**：根据元数据自动创建数据库表结构
- **数据同步**：自动化的量化数据同步流程
- **数据管理**：统一的数据存储和管理
- **数据分析**：内置数据分析能力
- **数据可视化**：提供数据可视化功能

## 技术架构

本项目基于 Go 开发，是之前 Python 版本的重构升级。核心架构特点：

- **工作流抽象**：通过 [task-engine](https://github.com/your-org/task-engine) 将复杂的元数据爬取解析、建表和数据同步过程抽象成工作流
- **并发管理**：由 task-engine 统一管理并发调度，简化开发工作
- **业务聚焦**：开发者只需定义业务方法和工作流配置，其余由 task-engine 自动调度

## 架构优势

相比之前的 Python 版本，本次重构带来了以下改进：

- ✅ **清晰的抽象层次**：业务逻辑与并发调度分离
- ✅ **简化开发**：只需关注业务方法和工作流配置
- ✅ **更好的可维护性**：工作流驱动的架构更易理解和维护
- ✅ **自动化调度**：task-engine 自动处理并发和调度
