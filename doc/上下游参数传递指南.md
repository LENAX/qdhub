## 1. 概述

在 Task Engine 中，任务之间的数据传递是通过 **结果缓存机制** 实现的。当一个任务执行完成后，其返回的结果会被引擎缓存，并自动注入到依赖它的下游任务的参数中。

### 参数传递流程

```
┌─────────────┐    结果缓存    ┌─────────────┐    结果缓存    ┌─────────────┐
│   Task A    │ ──────────▶  │   Task B    │ ──────────▶  │   Task C    │
│  (上游任务)  │              │  (模板任务)  │              │  (下游任务)  │
└─────────────┘              └─────────────┘              └─────────────┘
      │                            │
      │                            │ 生成子任务
      │                            ▼
      │                      ┌───────────┐
      │                      │ SubTask 1 │──┐
      │                      ├───────────┤  │ 结果聚合
      │                      │ SubTask 2 │──┼──────▶ Task B 结果
      │                      ├───────────┤  │
      │                      │ SubTask N │──┘
      │                      └───────────┘
      │
      └─────── Task C 可获取 Task A 和 Task B 的结果
```

### 两种结果类型

1. **上游任务结果**：直接依赖的前置任务返回的数据
2. **子任务结果**：模板任务生成的子任务执行后的聚合结果

## 2. 上游任务结果获取

### 2.1 基本概念

当任务 B 依赖任务 A 时，任务 A 执行完成后，其返回结果会被自动注入到任务 B 的 `TaskContext.Params` 中，使用以下两种 key：

- `_cached_<TaskID>` - 使用任务的 UUID
- `_cached_<TaskName>` - 使用任务的名称

> **推荐使用任务名称**：任务名称更具可读性，且在 Workflow 定义时已知。

### 2.2 获取上游任务结果

#### 方式一：使用新 API（推荐）

```go
func ProcessData(tc *task.TaskContext) (interface{}, error) {
    // 获取整个上游任务结果
    result := tc.GetUpstreamResult("FetchStockList")
    // result = map[string]interface{}{"stock_codes": [...], "count": 100}
    
    // 直接获取特定字段（类型安全）
    stockCodes := tc.GetUpstreamStringSlice("FetchStockList", "stock_codes")
    count, err := tc.GetUpstreamInt("FetchStockList", "count")
    source := tc.GetUpstreamString("FetchStockList", "source")
    
    // 处理数据...
    return result, nil
}
```

#### 方式二：使用旧方式（兼容）

```go
func ProcessData(tc *task.TaskContext) (interface{}, error) {
    // 手动获取缓存参数
    cached := tc.GetParam("_cached_FetchStockList")
    if cached != nil {
        if resultMap, ok := cached.(map[string]interface{}); ok {
            stockCodes := resultMap["stock_codes"].([]string)
        }
    }
    return nil, nil
}
```

### 2.3 API 参考

| 方法签名 | 说明 | 返回值 |
|---------|------|-------|
| `GetUpstreamResult(taskName string)` | 获取指定上游任务的完整结果 | `map[string]interface{}` |
| `GetAllUpstreamResults()` | 获取所有上游任务的结果 | `map[string]map[string]interface{}` |
| `GetUpstreamValue(taskName, field string)` | 获取上游结果的指定字段 | `interface{}` |
| `GetUpstreamString(taskName, field string)` | 获取字符串类型字段 | `string` |
| `GetUpstreamInt(taskName, field string)` | 获取整数类型字段 | `(int, error)` |
| `GetUpstreamStringSlice(taskName, field string)` | 获取字符串切片字段 | `[]string` |

### 2.4 使用示例

#### 示例 1：基本参数传递

```go
// 上游任务：获取股票列表
func FetchStockList(tc *task.TaskContext) (interface{}, error) {
    stocks := []string{"000001.SZ", "600000.SH", "601398.SH"}
    return map[string]interface{}{
        "stock_codes": stocks,
        "count":       len(stocks),
        "source":      "tushare",
    }, nil
}

// 下游任务：处理股票数据
func ProcessStocks(tc *task.TaskContext) (interface{}, error) {
    // 使用新 API 获取上游结果
    stockCodes := tc.GetUpstreamStringSlice("FetchStockList", "stock_codes")
    source := tc.GetUpstreamString("FetchStockList", "source")
    
    fmt.Printf("从 %s 获取了 %d 只股票\n", source, len(stockCodes))
    
    // 处理逻辑...
    return map[string]interface{}{
        "processed": len(stockCodes),
    }, nil
}
```

#### 示例 2：多上游任务

```go
// 任务 C 依赖 A 和 B
func TaskC(tc *task.TaskContext) (interface{}, error) {
    // 获取所有上游结果
    allResults := tc.GetAllUpstreamResults()
    
    for taskName, result := range allResults {
        fmt.Printf("任务 %s 的结果: %v\n", taskName, result)
    }
    
    // 分别获取 A 和 B 的结果
    resultA := tc.GetUpstreamResult("TaskA")
    resultB := tc.GetUpstreamResult("TaskB")
    
    return map[string]interface{}{
        "from_a": resultA["data"],
        "from_b": resultB["data"],
    }, nil
}
```

## 3. 子任务结果获取

### 3.1 基本概念

**模板任务** 是一种特殊任务，它在执行时动态生成多个 **子任务**。当所有子任务执行完成后，引擎会将子任务的结果聚合到模板任务的结果中，供下游任务使用。

#### 子任务结果结构

子任务结果以数组形式存储在模板任务的 `_cached_` 参数中：

```go
// 结构示意
_cached_TemplateTask = map[string]interface{}{
    "subtask_results": []map[string]interface{}{
        {
            "task_id":   "uuid-1",
            "task_name": "subtask-1",
            "status":    "Success",
            "result":    map[string]interface{}{...},  // 子任务返回值
        },
        {
            "task_id":   "uuid-2",
            "task_name": "subtask-2",
            "status":    "Failed",
            "error":     "错误信息",
        },
        // ...更多子任务结果
    },
}
```

### 3.2 获取子任务结果

#### SubTaskResult 结构体

```go
type SubTaskResult struct {
    TaskID   string                 // 子任务 UUID
    TaskName string                 // 子任务名称
    Status   string                 // 执行状态: "Success" | "Failed"
    Result   map[string]interface{} // 子任务返回的结果
    Error    string                 // 错误信息（仅失败时有值）
}

// 辅助方法
func (r *SubTaskResult) IsSuccess() bool                                   // 判断是否成功
func (r *SubTaskResult) GetResultValue(field string) interface{}           // 获取结果字段
func (r *SubTaskResult) GetResultMap(field string) map[string]interface{}  // 获取 map 类型字段
```

#### 获取子任务结果

```go
func AggregateResults(tc *task.TaskContext) (interface{}, error) {
    // 获取所有子任务结果
    allResults := tc.GetSubTaskResults()
    
    // 获取成功的子任务结果
    successResults := tc.GetSuccessfulSubTaskResults()
    
    // 获取失败的子任务结果
    failedResults := tc.GetFailedSubTaskResults()
    
    // 检查是否全部成功
    if tc.AllSubTasksSucceeded() {
        fmt.Println("所有子任务执行成功")
    }
    
    // 获取子任务总数
    count := tc.GetSubTaskCount()
    fmt.Printf("共 %d 个子任务，成功 %d，失败 %d\n", 
        count, len(successResults), len(failedResults))
    
    return nil, nil
}
```

### 3.3 API 参考

#### 子任务结果获取

| 方法签名 | 说明 | 返回值 |
|---------|------|-------|
| `GetSubTaskResults()` | 获取所有子任务结果 | `[]SubTaskResult` |
| `GetSuccessfulSubTaskResults()` | 获取成功的子任务结果 | `[]SubTaskResult` |
| `GetFailedSubTaskResults()` | 获取失败的子任务结果 | `[]SubTaskResult` |
| `AllSubTasksSucceeded()` | 检查是否全部成功 | `bool` |
| `GetSubTaskCount()` | 获取子任务总数 | `int` |

#### 从子任务提取数据

| 方法签名 | 说明 | 返回值 |
|---------|------|-------|
| `ExtractFromSubTasks(field string)` | 从成功子任务提取指定字段 | `[]interface{}` |
| `ExtractMapsFromSubTasks(field string)` | 提取 map 类型字段 | `[]map[string]interface{}` |
| `ExtractStringsFromSubTasks(field string)` | 提取字符串类型字段 | `[]string` |

### 3.4 使用示例

#### 示例 1：模板任务生成子任务

```go
// 模板任务：为每只股票生成数据获取子任务
func FetchDailyTemplate(tc *task.TaskContext) (interface{}, error) {
    // 从上游获取股票列表
    stockCodes := tc.GetUpstreamStringSlice("FetchStockList", "stock_codes")
    
    // 获取 InstanceManager 添加子任务
    type ManagerInterface interface {
        AtomicAddSubTasks(subTasks []types.Task, parentTaskID string) error
    }
    manager := tc.GetInstanceManager().(ManagerInterface)
    registry := eng.GetRegistry()
    
    // 为每只股票生成子任务
    subTasks := make([]types.Task, 0, len(stockCodes))
    for i, code := range stockCodes {
        subTask, _ := builder.NewTaskBuilder(
            fmt.Sprintf("fetch-daily-%d", i),
            fmt.Sprintf("获取 %s 日线数据", code),
            registry,
        ).
            WithJobFunction("FetchDailySub", map[string]interface{}{
                "stock_code": code,
            }).
            Build()
        subTasks = append(subTasks, subTask)
    }
    
    // 批量添加子任务
    manager.AtomicAddSubTasks(subTasks, tc.TaskID)
    
    return map[string]interface{}{
        "generated_count": len(subTasks),
    }, nil
}

// 子任务：获取单只股票日线数据
func FetchDailySub(tc *task.TaskContext) (interface{}, error) {
    stockCode := tc.GetParamString("stock_code")
    
    // 调用 API 获取数据...
    dailyData := fetchFromAPI(stockCode)
    
    return map[string]interface{}{
        "stock_code":  stockCode,
        "daily_data":  dailyData,
        "record_count": len(dailyData),
    }, nil
}
```

#### 示例 2：聚合子任务结果

```go
// 下游任务：聚合所有日线数据
func AggregateDailyData(tc *task.TaskContext) (interface{}, error) {
    // 检查子任务执行情况
    if !tc.AllSubTasksSucceeded() {
        failedResults := tc.GetFailedSubTaskResults()
        for _, r := range failedResults {
            log.Printf("子任务 %s 失败: %s", r.TaskName, r.Error)
        }
        // 决定是否继续处理成功的部分
    }
    
    // 从子任务提取日线数据
    dailyDataMaps := tc.ExtractMapsFromSubTasks("daily_data")
    
    // 统计信息
    totalRecords := 0
    for _, data := range dailyDataMaps {
        if count, ok := data["record_count"].(int); ok {
            totalRecords += count
        }
    }
    
    fmt.Printf("聚合了 %d 只股票的日线数据，共 %d 条记录\n", 
        len(dailyDataMaps), totalRecords)
    
    return map[string]interface{}{
        "aggregated_count": len(dailyDataMaps),
        "total_records":    totalRecords,
    }, nil
}
```

#### 示例 3：处理部分失败场景

```go
func HandlePartialFailure(tc *task.TaskContext) (interface{}, error) {
    successResults := tc.GetSuccessfulSubTaskResults()
    failedResults := tc.GetFailedSubTaskResults()
    
    // 记录失败的子任务
    failedCodes := make([]string, 0)
    for _, r := range failedResults {
        stockCode := r.GetResultValue("stock_code")
        if code, ok := stockCode.(string); ok {
            failedCodes = append(failedCodes, code)
        }
        log.Printf("❌ 股票 %s 获取失败: %s", stockCode, r.Error)
    }
    
    // 处理成功的子任务结果
    successCodes := tc.ExtractStringsFromSubTasks("stock_code")
    
    return map[string]interface{}{
        "success_count": len(successResults),
        "failed_count":  len(failedResults),
        "failed_codes":  failedCodes,
        "success_codes": successCodes,
    }, nil
}
```

## 4. 完整示例

以下是一个完整的股票数据处理管道示例：

```go
package main

import (
    "context"
    "fmt"
    
    "github.com/LENAX/task-engine/pkg/core/builder"
    "github.com/LENAX/task-engine/pkg/core/engine"
    "github.com/LENAX/task-engine/pkg/core/task"
    "github.com/LENAX/task-engine/pkg/core/types"
    "github.com/LENAX/task-engine/pkg/core/workflow"
)

func main() {
    // 1. 创建 Engine
    eng, _ := engine.NewEngine(10, 30, workflowRepo, instanceRepo, taskRepo)
    ctx := context.Background()
    eng.Start(ctx)
    defer eng.Stop()
    
    registry := eng.GetRegistry()
    
    // 2. 注册 Job Functions
    registry.Register(ctx, "FetchStockList", FetchStockList, "获取股票列表")
    registry.Register(ctx, "FetchDailyTemplate", FetchDailyTemplate, "生成日线子任务")
    registry.Register(ctx, "FetchDailySub", FetchDailySub, "获取单只股票日线")
    registry.Register(ctx, "AggregateData", AggregateData, "聚合数据")
    
    // 3. 构建 Workflow
    wf := workflow.NewWorkflow("stock-pipeline", "股票数据处理管道")
    
    task1, _ := builder.NewTaskBuilder("FetchStockList", "获取股票列表", registry).
        WithJobFunction("FetchStockList", nil).
        Build()
    
    task2, _ := builder.NewTaskBuilder("FetchDailyData", "获取日线数据", registry).
        WithJobFunction("FetchDailyTemplate", nil).
        WithDependency("FetchStockList").
        WithTemplate(true).  // 标记为模板任务
        Build()
    
    task3, _ := builder.NewTaskBuilder("AggregateData", "聚合数据", registry).
        WithJobFunction("AggregateData", nil).
        WithDependency("FetchDailyData").
        Build()
    
    wf.AddTask(task1)
    wf.AddTask(task2)
    wf.AddTask(task3)
    
    // 4. 提交执行
    controller, _ := eng.SubmitWorkflow(ctx, wf)
    
    // 5. 等待完成
    for {
        status, _ := controller.GetStatus()
        if status == "Success" || status == "Failed" {
            fmt.Printf("Workflow 完成，状态: %s\n", status)
            break
        }
        time.Sleep(time.Second)
    }
}

// Job Functions 实现
func FetchStockList(tc *task.TaskContext) (interface{}, error) {
    return map[string]interface{}{
        "stock_codes": []string{"000001.SZ", "600000.SH", "601398.SH"},
        "count":       3,
    }, nil
}

func FetchDailyTemplate(tc *task.TaskContext) (interface{}, error) {
    stockCodes := tc.GetUpstreamStringSlice("FetchStockList", "stock_codes")
    
    type ManagerInterface interface {
        AtomicAddSubTasks(subTasks []types.Task, parentTaskID string) error
    }
    manager := tc.GetInstanceManager().(ManagerInterface)
    registry := tc.GetDependency("Registry").(*plugin.Registry)
    
    subTasks := make([]types.Task, 0)
    for i, code := range stockCodes {
        subTask, _ := builder.NewTaskBuilder(
            fmt.Sprintf("fetch-daily-%d", i),
            fmt.Sprintf("获取 %s 日线", code),
            registry,
        ).
            WithJobFunction("FetchDailySub", map[string]interface{}{
                "stock_code": code,
            }).
            Build()
        subTasks = append(subTasks, subTask)
    }
    
    manager.AtomicAddSubTasks(subTasks, tc.TaskID)
    return map[string]interface{}{"generated": len(subTasks)}, nil
}

func FetchDailySub(tc *task.TaskContext) (interface{}, error) {
    code := tc.GetParamString("stock_code")
    return map[string]interface{}{
        "stock_code": code,
        "daily_data": map[string]interface{}{
            "open": 10.5, "close": 10.8, "high": 11.0, "low": 10.2,
        },
    }, nil
}

func AggregateData(tc *task.TaskContext) (interface{}, error) {
    // 使用新 API 获取子任务结果
    count := tc.GetSubTaskCount()
    allSucceeded := tc.AllSubTasksSucceeded()
    dailyDataMaps := tc.ExtractMapsFromSubTasks("daily_data")
    
    fmt.Printf("聚合 %d 只股票数据，全部成功: %v\n", count, allSucceeded)
    
    return map[string]interface{}{
        "aggregated": len(dailyDataMaps),
        "success":    allSucceeded,
    }, nil
}
```

## 5. 最佳实践

### 5.1 任务命名

- 使用清晰、有意义的任务名称
- 任务名称应能反映任务的功能
- 避免使用特殊字符

```go
// ✅ 好的命名
"FetchStockList"
"ProcessDailyData"
"SaveToDatabase"

// ❌ 不好的命名
"Task1"
"Process_Data_123"
"fetch-stock@list"
```

### 5.2 结果返回格式

- 始终返回 `map[string]interface{}` 类型
- 使用有意义的字段名
- 包含足够的上下文信息

```go
// ✅ 好的返回格式
return map[string]interface{}{
    "stock_codes": stockCodes,
    "count":       len(stockCodes),
    "source":      "tushare",
    "fetch_time":  time.Now().Format(time.RFC3339),
}, nil

// ❌ 不好的返回格式
return stockCodes, nil  // 丢失上下文
```

### 5.3 错误处理

- 子任务失败不会自动终止整个 Workflow
- 下游任务应检查子任务执行状态
- 记录失败信息便于排查

```go
func ProcessResults(tc *task.TaskContext) (interface{}, error) {
    if !tc.AllSubTasksSucceeded() {
        failedResults := tc.GetFailedSubTaskResults()
        for _, r := range failedResults {
            log.Printf("子任务 %s 失败: %s", r.TaskName, r.Error)
        }
        
        // 根据业务逻辑决定是否继续
        if len(failedResults) > threshold {
            return nil, fmt.Errorf("失败子任务过多: %d", len(failedResults))
        }
    }
    
    // 只处理成功的结果
    successResults := tc.GetSuccessfulSubTaskResults()
    // ...
}
```

### 5.4 性能考虑

- 大量子任务时，使用 `AtomicAddSubTasks` 批量添加
- 避免在循环中重复调用 `GetUpstreamResult`
- 使用 `Extract*FromSubTasks` 方法而非手动遍历

```go
// ✅ 高效方式
dailyDataMaps := tc.ExtractMapsFromSubTasks("daily_data")

// ❌ 低效方式
results := tc.GetSubTaskResults()
for _, r := range results {
    data := r.GetResultMap("daily_data")
    // 每次调用都有类型检查开销
}
```
