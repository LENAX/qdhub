# QDHub 设计

基于task engine实现一个量化数据管理系统，主要有元数据管理、自动建表、数据同步、分析和可视化功能。数据源主要使用tushare,但也可以被扩展，支持akshare等数据源。获得的元数据可以被用于自动建表和数据同步的参数自动注入。数据同步支持跑批更新和实时更新。量化数据可以被存在多种数据库上，但主要支持duckdb, 后面会扩展到postgres和clickhouse等数据库上。系统数据库支持sqlite, mysql和postgres。

系统支持的操作：

1. 元数据管理
   1. 初始化/刷新：爬取数据源的文档，获取api目录，api详细信息，api请求参数、api返回的数据字段
      1. 通常仅需一次
   2. 查看：获取目录、api信息
   3. token管理：用户可以设定token(加密)
2. 量化数据库管理
   1. 自动建表：当元数据完成初始化后，可以指定数据库类型（目前只支持duckdb), 存储目录/dsn, 根据元数据完成建表
      1. 通常仅需一次
3. 量化数据管理
   1. 自动同步（批量模式)
      1. 用户可以选定要同步的数据源和api，指定更新的频率，系统会自动在规定的时间启动同步
   2. 实时同步
      1. 系统会给出可以实时同步的数据，设定的时间段，或者可以24小时采集，用户选择同步哪些
   3. 同步状态监控
      1. 查看同步工作流的历史和当前运行的工作流的状态和进展
      2. 可以暂停/重启工作流
   4. 数据可视化
      1. 同步完成的数据，可以查询，或者在前端可视化
4. 工作流管理和配置
   1. 用户可以编辑/查看/上传工作流，可以启用或者禁用工作流
   2. 用户可以查看工作流实例的运行状态/历史，管理实例的运行状态(终止，暂停，重启，断点续传)

技术要求：

1. http api: Gin
2. 数据库: sqlite(开发)，postgres(生产)
3. 数据库交互: sqlx
4. 工作流管理: github.com/stevelan1995/task-engine
5. 数据源api接口
   1. tushare,没有go实现，参考python包复刻一个。https://tushare.pro/document/1?doc_id=130
   2. akshare, 没有go实现，以后支持
6. 前端：vue + nativeui (已经有成品)
7. 架构：轻量ddd(业务模型, repository), 基于接口松耦合，依赖倒置，依赖注入
